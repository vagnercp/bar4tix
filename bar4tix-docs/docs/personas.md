# 👥 Personas & Jornadas — Bar4tix (v0.1)

## 🧭 Propósito

As personas e jornadas descrevem os perfis centrais que interagem com o ecossistema **Bar4tix**, orientando decisões de produto, UX, dados e monetização.
Baseadas em dados de consumo, estudos de comportamento e hipóteses validadas no MVP.

---

## 🧍‍♀️ Persona 1 — **Consumidor Econômico (Ana, 32)**

**Segmento:** B2C — App/Web
**Perfil:** mulher, 32 anos, renda média, busca economia e conveniência.

### 🎯 Objetivos

* Comparar preços de produtos da cesta básica e farmácia.
* Receber alertas de queda de preço em mercados próximos.
* Entender o impacto das promoções semanais no orçamento.

### 😣 Dores

* Falta de transparência nos preços entre redes.
* Aplicativos de ofertas com dados desatualizados.
* Dificuldade em organizar listas e preferências.

### 💡 O que valoriza

* Confiabilidade e atualizações rápidas.
* Alertas automáticos (push) baseados em check-in geolocalizado.
* Programas de cashback e recompensas.

### 📊 Indicadores de Sucesso

| Métrica                          | Meta           |
| -------------------------------- | -------------- |
| Tempo médio de uso semanal       | ≥ 10 min       |
| Taxa de QR codes escaneados      | ≥ 1 por semana |
| Engajamento com alertas de preço | ≥ 40% clique   |
| NPS (satisfação app)             | ≥ 75           |

### 🔄 Jornada

1. Scan do QR da NFC-e no app.
2. Sistema identifica produtos e atualiza o comparador.
3. App sugere mercados próximos com menor preço.
4. Usuária adiciona produtos à lista e ativa alertas.
5. Recebe notificações quando há promoções relevantes.

## 👨‍💼 Persona 2 — **Gerente de Loja (Carlos, 41)**

**Segmento:** B2B — Lojistas/Redes
**Perfil:** gerente regional de supermercado com múltiplas filiais.

### 🎯 Objetivos

* Monitorar concorrência e ajustar promoções locais.
* Otimizar mix de produtos e margens por categoria.
* Engajar clientes via campanhas geolocalizadas.

### 😣 Dores

* Falta de visão consolidada do mercado regional.
* Comparadores com dados incompletos ou imprecisos.
* Processos manuais de atualização de preço.

### 💡 O que valoriza

* Dashboards de concorrência e BI em tempo real.
* Exportação para ERP/CRM.
* Alertas automáticos de variação de preço.

### 📊 Indicadores de Sucesso

| Métrica                        | Meta     |
| ------------------------------ | -------- |
| Lojas ativas na plataforma     | ≥ 10     |
| Redução média de ruptura       | ≤ 5%     |
| Aumento de margem líquida      | ≥ 3 p.p. |
| Tempo de atualização de preços | ≤ 2 dias |

### 🔄 Jornada

1. Cadastro e verificação de CNPJ/filiais.
2. Conexão com ERP ou planilha.
3. Acesso a painéis de BI e comparador regional.
4. Recebe alertas de concorrência e variações.
5. Exporta dados e ajusta preços/promos localmente.

## 👩‍💻 Persona 3 — **Analista de Trade Marketing (Marina, 29)**

**Segmento:** B2B — Indústria/Distribuidor
**Perfil:** analista de trade de bens de consumo, foco em elasticidade e sortimento.

### 🎯 Objetivos

* Entender elasticidade e comportamento regional de SKUs.
* Medir participação de marca e share de gôndola.
* Integrar insights a relatórios de performance.

### 😣 Dores

* Dados inconsistentes ou não granularizados.
* Falta de histórico regionalizado por SKU.
* Dificuldade em cruzar preço, demanda e estoque.

### 💡 O que valoriza

* API de dados limpa e versionada.
* Relatórios de elasticidade por categoria/região.
* Acesso rápido a histórico e gráficos de preço médio.

### 📊 Indicadores de Sucesso

| Métrica                             | Meta    |
| ----------------------------------- | ------- |
| Cobertura de SKU nas análises       | ≥ 95%   |
| Tempo médio de geração de relatório | ≤ 2 min |
| Satisfação do time de trade         | ≥ 90%   |
| Elasticidade calculada por SKU top  | ≥ 70%   |

### 🔄 Jornada

1. Acessa portal B2B e assina relatórios.
2. Configura filtros (região, categoria, SKU).
3. Gera visualizações e insights automáticos.
4. Exporta relatórios para ERP/BI interno.
5. Alimenta decisões de pricing e promoções.

## 🧑‍💼 Persona 4 — **Gestor Público (Rafael, 45)**

**Segmento:** B2G — Governos e Instituições
**Perfil:** servidor municipal/estadual, responsável por indicadores econômicos e políticas públicas.

### 🎯 Objetivos

* Monitorar o custo da cesta básica municipal.
* Mapear variações de preço entre bairros e estabelecimentos.
* Identificar impactos inflacionários locais.

### 😣 Dores

* Dados oficiais defasados e coleta manual.
* Falta de API aberta com granularidade suficiente.
* Dificuldade em comunicar insights à população.

### 💡 O que valoriza

* Painel regional “Cesta Bar4tix”.
* API aberta de dados anonimizados.
* Dashboards integráveis a portais públicos.

### 📊 Indicadores de Sucesso

| Métrica                              | Meta     |
| ------------------------------------ | -------- |
| Atualizações de índice por semana    | ≥ 1      |
| Tempo médio de coleta → publicação   | ≤ 48h    |
| Acurácia dos dados                   | ≥ 97%    |
| Uso de API pública (requisições/mês) | ≥ 10.000 |

### 🔄 Jornada

1. Assina convênio de dados públicos (Bar4tix Data Open).
2. Define regiões e categorias de acompanhamento.
3. Gera painéis com índices de inflação e variação média.
4. Divulga “Cesta Bar4tix” nos canais oficiais.
5. Usa insights em políticas de combate à carestia.
